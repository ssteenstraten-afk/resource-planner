bezetting

Kernentiteit: hoeveel uur werkt een consultant in een bepaalde week aan een project.

KolomTypeBeschrijving
consultant_iduuidVerwijzing naar consultants
project_iduuidVerwijzing naar projecten
jaarintegerISO jaarnummer
weekintegerISO weeknummer (1–53)
urennumeric(4,1)Uren die week (0–60, uniek per combinatie)

bezetting_snapshot / bezetting_log

bezetting_snapshot — Wekelijkse momentopname van alle toekomstige bezetting (aangemaakt elke zondag om 23:00). Gebruikt voor historische vergelijking.

bezetting_log — Automatische auditlog: bij elke wijziging in bezetting wordt via een PostgreSQL trigger een rij aangemaakt met oude en nieuwe waarde.

Berekening beschikbaarheid

Bezette uren = som van alle bezetting.uren voor een consultant in een week

Beschikbare uren = contract_uren − bezette uren

Bezettingsgraad = (bezette uren / contract_uren) × 100%

BezettingsgraadStatusKleur
< 60%OnderbezetRood
60–79%GemiddeldOranje
≥ 80%Goed bezetGroen
> 100%OverbezetPaars
Functionele specificatie

Use Cases

Consultant

1
Inloggen via magic link of wachtwoord
Consultant

De consultant ontvangt wekelijks een e-mail met een directe inloglink, of logt handmatig in via het loginscherm.

  1. Consultant opent de app of klikt op de link in de maandagmail
  2. Bij handmatig inloggen: vul e-mailadres en wachtwoord in, of kies "magic link"
  3. Na succesvolle authenticatie wordt doorgestuurd naar /mijn-week
2
Weekoverzicht bekijken
Consultant

De consultant ziet een tabel met hun bezetting voor de komende 12 weken, per project per week.

  1. Consultant navigeert naar /mijn-week
  2. Tabel toont rijen per toegewezen project, kolommen per week
  3. Onderaan: totaal bezette uren, contracturen en beschikbaarheid per week
  4. Kleurcodering geeft snel inzicht in bezettingsgraad
3
Uren aanpassen
Consultant

De consultant kan uren per project per week inline bewerken.

  1. Klik op een cel in het weekoverzicht
  2. Voer het nieuwe aantal uren in (0–60)
  3. Bevestig met Enter of klik ergens anders
  4. Gewijzigde cellen worden geel gemarkeerd
  5. Klik "Bevestig wijzigingen" om op te slaan
4
Validatie bij invoer
Consultant

Het systeem voorkomt ongeldige invoer.

  1. Weeksom > 60 uur → opslaan geblokkeerd met foutmelding
  2. Waarde > 12 uur in één cel → waarschuwing (maar niet geblokkeerd)
  3. Overbezette weken worden paars gemarkeerd in de footer
  4. Server-side CHECK constraint als extra beveiliging
5
Verlof en bench registreren
Consultant

Systeemprojecten "Verlof" en "Bench" zijn altijd zichtbaar en bewerkbaar.

  1. Consultant vult uren in bij "Verlof" of "Bench" rij
  2. Deze tellen mee in de bezettingsberekening
  3. Systeemprojecten kunnen niet worden verwijderd door de planner
6
Wekelijkse bezettingsemail ontvangen
Consultant

Elke maandag om 07:00 ontvangt de consultant automatisch een overzicht per e-mail.

  1. Supabase Edge Function genereert email voor elke actieve consultant
  2. Email bevat tabel met bezetting komende 12 weken
  3. Kleurcodering op basis van bezettingsgraad
  4. "Bekijk en pas aan" knop bevat een directe magic link naar /mijn-week

Planner

7
Bezettingsoverzicht bekijken (heatmap)
Planner

De planner heeft een overzicht van alle consultants met hun bezettingsgraad per week.

  1. Planner navigeert naar /dashboard
  2. Heatmap toont alle actieve consultants × komende 12 weken
  3. Cellen zijn gekleurd op basis van bezettingsgraad (rood/oranje/groen/paars)
  4. Klik op een cel toont tooltip met projectuitsplitsing
  5. Weeknavigatie om te schuiven naar andere periodes
8
Filteren en zoeken
Planner

De planner kan de heatmap filteren op functieniveau en specifieke consultant.

  1. Dropdown "Alle niveaus" filtert op functieniveau
  2. Dropdown "Alle consultants" filtert op individuele consultant
  3. Filters zijn combineerbaar
9
Consultant detail bekijken
Planner

De planner kan inzoomen op de bezetting van één consultant.

  1. Klik op een naam in de heatmap
  2. Navigeert naar /dashboard/consultant/:id
  3. Zelfde tabelindeling als /mijn-week, maar read-only
  4. Toont naam, functieniveau en contracturen bovenaan
10
Project aanmaken
Planner

De planner maakt nieuwe projecten aan in het systeem.

  1. Navigeer naar /dashboard/projecten
  2. Klik "+ Nieuw project"
  3. Vul naam (verplicht), klant, start- en einddatum in
  4. Project wordt aangemaakt met status 'actief'
11
Consultant toewijzen aan project
Planner

De planner wijst consultants toe aan projecten. De consultant ziet het project daarna in hun weekoverzicht.

  1. Klik "Consultants (n)" bij een project
  2. Selecteer een consultant uit de dropdown
  3. Klik "Toewijzen"
  4. Systeem maakt bezettingsrecords aan voor alle weken in de projectperiode (0 uur)
  5. Consultant ziet het project nu in /mijn-week en kan uren invullen
  6. Toewijzing verwijderen via × knop bij de consultant
12
Project afsluiten
Planner

Afgesloten projecten blijven zichtbaar in de lijst maar verdwijnen uit het weekoverzicht van consultants.

  1. Klik "Afsluiten" bij een actief project
  2. Status wijzigt naar 'afgesloten'
  3. Project kan worden heropend via "Heropenen"
  4. Systeemprojecten (Verlof, Bench) kunnen niet worden afgesloten
13
Nieuwe gebruiker aanmaken
Planner

De planner maakt nieuwe gebruikersaccounts aan zonder het Supabase dashboard te hoeven gebruiken.

  1. Navigeer naar /dashboard/consultants
  2. Klik "+ Nieuwe gebruiker"
  3. Vul naam, e-mail, functieniveau, contracturen en rol in
  4. Systeem roept de invite-consultant Edge Function aan
  5. Supabase Auth account + consultant record worden aangemaakt
  6. Nieuwe gebruiker kan direct inloggen via magic link
14
Gebruiker deactiveren
Planner

De planner kan gebruikers deactiveren zodat ze niet meer kunnen inloggen maar historische data behouden blijft.

  1. Ga naar /dashboard/consultants
  2. Klik "Deactiveren" bij de betreffende gebruiker
  3. Gebruiker verdwijnt uit de heatmap en kan niet meer inloggen
  4. Historische bezettingsdata blijft bewaard
Automatisering

Edge Functions & Cron Jobs

A
weekly-snapshot — elke zondag 23:00
Automatisch

Maakt een wekelijkse momentopname van alle toekomstige bezettingsdata.

  1. Bepaal huidige ISO jaar en weeknummer
  2. Controleer of snapshot voor deze week al bestaat (idempotent)
  3. Kopieer alle bezettingsrijen vanaf huidige week naar bezetting_snapshot
  4. Gebruikt voor historische vergelijking van planningswijzigingen
B
send-weekly-emails — elke maandag 07:00
Automatisch

Verstuurt wekelijkse bezettingsoverzichten naar alle actieve consultants via Resend.

  1. Haal alle actieve consultants op
  2. Per consultant: bereken bezetting komende 12 weken
  3. Genereer persoonlijke magic link voor directe toegang
  4. Verstuur HTML email via Resend API met kleurgecodeerde tabel
C
invite-consultant — on demand
API Call

Wordt aangeroepen vanuit de planner-interface bij het aanmaken van een nieuwe gebruiker.

  1. Ontvangt naam, email, functieniveau, contracturen en rol
  2. Maakt Supabase Auth gebruiker aan (email bevestigd)
  3. Maakt consultant record aan in de database
  4. Bij fout: auth gebruiker wordt automatisch verwijderd (rollback)

Beveiliging

LaagMaatregel
AuthenticatieSupabase Auth — magic links of wachtwoord, geen plain-text opslag
AutorisatieRow Level Security op alle tabellen — consultant ziet alleen eigen data
Validatie (client)Weeksom max 60u, waarschuwing bij >12u per cel
Validatie (server)CHECK constraint op bezetting.uren (0–60)
Audit trailPostgreSQL trigger logt elke wijziging in bezetting_log
API keysService role key alleen in Supabase Secrets, nooit in de browser
